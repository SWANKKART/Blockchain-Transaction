## 📌 About

HashVault demonstrates how blockchain technology can secure financial transactions between peers — without a central authority like a bank. Every transaction is cryptographically hashed and chained to the previous block, making the ledger tamper-evident and transparent.

This project was built as a **Final Year B.Sc. Information Technology** project exploring the fundamentals of blockchain infrastructure using Python.

---

## 🧠 Key Concepts Demonstrated

| Concept | Description |
|---|---|
| **SHA3-256 Hashing** | Converts any transaction data into a unique 256-bit fingerprint |
| **Case Sensitivity** | Even `"HELLO"` vs `"hello"` produces a completely different hash |
| **Block Structure** | Each block holds data, a nonce, previous hash, and its own hash |
| **Immutability** | Changing any block's data changes its hash, breaking the entire chain |
| **Tamper Detection** | Any modification is immediately detectable by hash mismatch |

## 🔬 How It Works

### 1. Hashing a transaction

```python
import hashlib

m = hashlib.sha3_256()
m.update(b'Alice sends Bob 500 INR')
tx_hash = m.hexdigest()

print(f'Hash: {tx_hash}')
# Output: a unique 64-character hex string
```

### 2. Case sensitivity proves data integrity

```python
import hashlib

# Uppercase
m = hashlib.sha3_256()
m.update(b'HELLO SECURITY GUY')
hash1 = m.hexdigest()

# Lowercase — completely different hash!
n = hashlib.sha3_256()
n.update(b'hello security guy')
hash2 = n.hexdigest()

print(hash1)  # 2503a43f8234aa01...
print(hash2)  # 3436d160d007c7b5...
```

### 3. Block structure

```
Block #1
├── data      → "Alice → Bob : ₹500"
├── nonce     → 3847291
├── prev_hash → "0000a4f82b3c9e1d..."  ← links to Block #0
└── hash      → "00003d7c5a2f8b1e..."  ← unique fingerprint
```

If **any** value in the block is changed, the hash changes — breaking every block that follows it.
